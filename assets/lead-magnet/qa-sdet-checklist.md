# QA/SDET Resume and Portfolio Checklist

Use this checklist before applying to QA, automation QA, or SDET roles.

## Resume

- Is your target role obvious within the first few lines?
- Do your bullets describe testing impact, not just tasks?
- Have you named tools clearly, such as Selenium, Playwright, Postman, Cypress, JUnit, TestNG, or API testing frameworks?
- Did you quantify outcomes where possible, such as regression coverage, defect reduction, or release confidence?
- Does each project bullet show what you tested, how you tested it, and why it mattered?
- Did you remove generic wording that could apply to any candidate?

## LinkedIn

- Does your headline clearly mention QA, test automation, or SDET goals?
- Does your summary explain your strengths in quality engineering, communication, and debugging?
- Do your featured links point to relevant projects or GitHub repositories?

## Portfolio

- Do you have at least one recruiter-friendly project that can be understood in under 2 minutes?
- Does each repository include a clear README with setup steps and project purpose?
- Have you explained why you chose the test framework or tooling?
- Do your projects show more than happy-path checks?
- Do you include API, UI, or end-to-end coverage where relevant?
- Is your folder structure clean enough for another engineer to browse quickly?

## Interview Readiness

- Can you explain how you design test cases for ambiguous features?
- Can you describe a bug clearly with steps, expected result, actual result, and severity?
- Can you explain the difference between manual QA, automation QA, and SDET responsibilities?
- Do you have at least three strong stories about debugging, collaboration, and test strategy?
- Can you explain one testing project from goal through implementation and tradeoffs?
